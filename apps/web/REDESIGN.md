### Замороженный контракт (конкретика, сверено с motion.dev)
- **Тема — только тёмная.** Подъём = ТЕМНЕЕ (`bg #0d1111` → `surface #060607` → `surface-2 #030304`);
  лифт несут мягкие тени `--shadow-1..4`, не цвет.
- **Текст** `#ededec` / **muted** `#7a8180` / **faint** `#525a59`. **Границы** 1px: `border #1e2427`,
  `border-strong #2b3338`.
- **Акцент — ПРОВИЗОРНЫЙ** мятный `#8df0cc` (живёт за `--color-accent*`, меняется в Фазе 2 без правки
  компонентов). Никогда не хардкодить hex в компонентах — только токены.
- **Радиусы — две семьи:** угловатое = `rounded-none` (0), круглое = `rounded-full`; `--radius-sm` 3px
  только для полей/кода. Средний радиус 8–16px НЕ используем (это и есть «острый» вид).
- **Подпись:** угловатые интерактивы носят `.cornerframe` (+`.cornerframe-fill` для заливки) —
  4 уголка-скобки, на hover/focus выезжают наружу, на active вжимаются; диагональная заливка-вытирание;
  моно-капс лейбл `label-mono` инвертируется. Реализация — ОДНА (`index.css`), агенты не пишут свою.
- **Круглая семья:** `IconButton` (round), аватары/чипы/индикаторы.
- **Штриховка** `hatch`/`hatch-strong`/`hatch-accent` — диагональ **119°** (фирменный угол), 1px/5px.
- **Стекло** — утилиты `glass`/`glass-strong`/`glass-badge` (+ непрозрачный фолбэк); дозированно.
- **Шрифты:** `--font-mono` = **Geist Mono** (несущий — все капс-лейблы), `--font-sans` = **Inter**
  (TASA Orbiter под лицензией → позже). `label-mono` = mono, uppercase, tracking .18em, 11px, wght 560.
- **Движение:** домашние кривые `--ease-out: cubic-bezier(.25,0,0,1)` (основная),
  `--ease-expo: cubic-bezier(.16,1,.3,1)`; токены `--dur-fast/dur/dur-slow`; всё уважает
  `prefers-reduced-motion`.
- **Иконки:** только `<Icon name size className/>` (lucide под капотом); `strokeWidth` заморожен = 1.75;
  без `image-rendering: pixelated`.
- **Легаси-алиасы** (`--color-twitch*`, `--color-line`, `--shadow-pixel*`, `card-pixel*`) временно
  указывают на новые значения, чтобы немигрированный код компилировался и смягчался — удалить по ходу Фазы 1.

## 0. Видение

Уходим от пиксель-арта «Midnight Arcade» к современному тёмному стилю.

- **База:** тёмная тема (только тёмная), эстетика уровня Vercel — тонкие границы, мягкие тени,
  аккуратное матовое **стекло**, дозированный, но смелый акцент.
- **Подпись бренда (отличие от «просто Vercel»):**
  - **Угловатое семейство** — прямоугольные интерактивные элементы (кнопки, карточки, поля) с
    тонкой границей и **анимацией уголков при наведении/фокусе** (corner accents). Реализация
    угловой анимации — ОДНА общая (см. §2.6), не выдумывается каждым агентом.
  - **Круглое семейство** — парные полностью скруглённые элементы (icon-кнопки, аватары,
    тоглы, чипы-статусы, индикаторы). Мягкий контрапункт к угловатым.
  - Анимаций не стесняемся (фоновые, параллакс, курсор, hover-микровзаимодействия) — но всегда с
    `prefers-reduced-motion` фолбэком и в рамках перф-бюджета (§5).
- **Иконки:** Lucide (`lucide-react`).
- **Анимации:** разрешена motion-библиотека (`motion`, бывш. Framer Motion).
- **Точные значения** (палитра, шрифты, радиусы, тайминги) подставляются под референсы заказчика —
  до этого в токенах стоят ПРОВИЗОРНЫЕ значения-заглушки (см. §6).

## 1. Фазы

Порядок исправлен относительно исходного: **сначала фундамент, потом фан-аут.** Иначе агенты
захардкодят случайные тени/радиусы/стекло.

### Фаза 0 — Фундамент (СЕРИЙНО, один контролируемый проход, замораживается до фан-аута)
Цель: создать «контракт» и эталоны, на которые опираются все остальные.
- Заменить `@theme` в `index.css` на новую токен-систему (§2.1–2.5): семантические цвета-роли,
  шкалы теней/радиусов, рецепт стекла, motion-токены. Значения — провизорные.
- Удалить пиксель-утилиты (`card-pixel*`, `scanlines`, `loader-board/tile`, `eq`, `seek-cells`,
  `mat-checker` оставить, `progress-indeterminate` переосмыслить) и `steps()`-кейфреймы.
- Подключить `lucide-react`; переписать `ui/icons.tsx` как тонкую обёртку над Lucide (сохранив
  API `<Icon name=... size=... className=... />`, чтобы не трогать все вызовы) ИЛИ постепенно
  заменить вызовы — решение в §5.1.
- Создать общий механизм **угловой анимации** (`CornerFrame`/утилита) и базовый **`Surface/Glass`**.
- Перерисовать 3–4 эталонных примитива: `Button`, `Card`, `Input`, `Surface` — как референс.
- Создать **галерею компонентов** — роут `/_gallery` (kitchen sink): все примитивы, плееры,
  состояния (hover/focus/disabled/loading/error) на одной странице. Инструмент само-верификации
  для агентов и визуального сравнения.
- **Критерий выхода:** галерея рендерит эталонные примитивы в новом стиле; `typecheck`+`lint`
  зелёные; контракт (§2) финализирован и заморожен.

### Фаза 1 — Реског существующих компонентов (ПАРАЛЛЕЛЬНО, фан-аут по компонентам)
Цель: перевести ВСЕ компоненты/страницы на новую систему. Структура и расположение — без изменений.
- Каждый агент берёт свой компонент из §3, применяет токены/примитивы/контракт, сверяется с галереей.
- Никаких пиксель-идиом: `image-rendering:pixelated`, `steps()`-анимации, жёсткие offset/inset-тени,
  `border-2 border-line`, `shadow-pixel*`/`card-pixel*`, пиксель-шрифты, `loader-board`/`scanlines`/`pixel-blink`.
  ВАЖНО: `rounded-none` — это УГЛОВАТЫЙ стиль системы (см. STATUS), НЕ пиксель-идиома; оставляем.
- **Критерий выхода:** ни одного `pixelated`/`steps(`/`shadow-pixel`/`border-2 border-line`/`text-twitch` в
  `apps/web/src`; все страницы визуально согласованы; `typecheck`+`lint` зелёные.

### Фаза 2 — Финализация бренда (СЕРИЙНО, дёшево)
Цель: подставить итоговые **шрифты** и **акцентную палитру** под референсы.
- Меняются ТОЛЬКО значения токенов в `index.css` (+ `@fontsource`/импорт шрифтов). Компоненты не трогаем.
- **Критерий выхода:** бренд утверждён, галерея и страницы выглядят финально по цвету/типографике.

### Фаза 3 — Перекомпоновка интерфейса (IA / лейаут)
Цель: изменить расположение/иерархию там, где захотим (не косметика, а структура экранов).
- Тонкие страницы + примитивы делают это безопасным. Объём определим отдельным под-планом.

### Фаза 4 — Анимации
Цель: плавные переходы, фоновые анимации, параллакс, курсор-эффекты, hover-микроанимации.
- На базе motion-токенов (§2.5) и выбранной библиотеки; везде `prefers-reduced-motion`; перф-бюджет (§5.3).

> Примечание по твоему исходному порядку: Фаза 1 (реског) РАНЬШЕ Фазы 2 (цвета/шрифты) работает
> только потому, что Фаза 0 выносит цвет/шрифт в токены-переменные — тогда Фаза 2 это смена значений,
> а не переделка компонентов.

## 2. Style Contract (обязателен для всех агентов)

> Значения ниже — ПРОВИЗОРНЫЕ заглушки до референсов. Архитектура (имена ролей/шкал) — финальная.

### 2.1 Цвета-роли (семантические, не «сырые» hex в компонентах)
- `--color-bg`, `--color-bg-elevated` — фон страницы / приподнятая поверхность.
- `--color-surface`, `--color-surface-2` — карточки/панели.
- `--color-border`, `--color-border-strong` — тонкие границы (1px по умолчанию).
- `--color-text`, `--color-muted`, `--color-faint` — текст по убыванию контраста.
- `--color-accent`, `--color-accent-hover`, `--color-accent-contrast` — акцент (значение в Фазе 2).
- Статусы: `--color-ok | warn | danger | info` (+ их `-soft` фоновые версии).
- Стекло: `--glass-bg` (rgba), `--glass-border` (rgba). См. §2.4.

### 2.2 Тени (мягкие, многослойные — НЕ пиксельные offset)
Шкала elevation: `--shadow-1` … `--shadow-4` (растёт мягкость/спред). Пример уровня 2:
`0 1px 2px rgba(0,0,0,.3), 0 4px 16px rgba(0,0,0,.25)`. Никаких `Npx Npx 0 0`.

### 2.3 Радиусы (две семьи) — ФРОЗЕН: см. STATUS
- Угловатая семья: `rounded-none` (0) — кнопки, карточки, фреймы. Это и есть «острый» вид motion.dev.
- Круглая семья: `rounded-full` (9999px) — icon-кнопки, аватары, чипы, тоглы, индикаторы.
- `--radius-sm` (3px) — ТОЛЬКО поля ввода/код. Средний радиус 8–16px НЕ используем (его отсутствие = look).

### 2.4 Стекло (один рецепт)
Класс/утилита `glass`: `background: var(--glass-bg)` + `backdrop-filter: blur(N) saturate(…)` +
`border: 1px solid var(--glass-border)` + `--shadow-2`. Использовать дозированно (модалки, поповеры,
плавающие панели, бар плеера) — НЕ на всём подряд (перф). Всегда иметь непрозрачный фолбэк, если
`backdrop-filter` не поддержан.

### 2.5 Движение (motion-токены)
- Длительности: `--dur-fast` (~120ms), `--dur` (~200ms), `--dur-slow` (~360ms).
- Easing: `--ease-out`, `--ease-in-out`, `--ease-spring` (для motion-библиотеки — пресет пружины).
- Правило: микровзаимодействия `--dur-fast`/`--ease-out`; входы/лейаут `--dur`/`--ease-in-out`.
- **Все** анимации обязаны иметь ветку `@media (prefers-reduced-motion: reduce)` / проверку
  `useReducedMotion()`.

### 2.6 Подпись: угловая анимация (ОДНА реализация)
Единый компонент-обёртка (рабочее имя `CornerFrame`) или утилита, дающая 4 уголка-акцента, которые
по `hover`/`focus-visible` «прорисовываются»/расширяются (тонкая линия акцентного цвета). Все угловатые
интерактивные элементы используют ЕЁ — агенты не пишут свои варианты. Параметры: толщина, длина уголка,
цвет (акцент по умолчанию), длительность (`--dur-fast`). Спека уголка финализируется в Фазе 0 под референс.

### 2.7 Фокус и доступность
- Видимый `focus-visible` ринг на всех интерактивах (через токен `--ring`), не обрезанный соседями.
- Контраст текста ≥ WCAG AA на стекле/мягких фонах (проверять, т.к. стекло снижает контраст).
- Хит-зоны ≥ 32px; `aria-*` сохраняем как в текущих компонентах.

### 2.8 Do / Don't
- DO: токены ролей, две семьи радиусов, общий `glass`, общий `CornerFrame`, Lucide-иконки, motion-токены.
- DON'T: сырые hex в компонентах, `image-rendering: pixelated`, `steps()`-анимации, жёсткие offset/inset-тени,
  `border-2 border-line`, `shadow-pixel*`/`card-pixel*`, неон-glow (кроме оверлея, см. §5.4), собственные ad-hoc
  корнер/стекло реализации. (`rounded-none` — это угловатый DO, НЕ запрет.)

## 4. Модель исполнения (ultracode)

- **Сериализуй фундамент.** Фаза 0 — НЕ параллельно. Токены, иконки, `CornerFrame`, `Surface` и
  эталонные примитивы — общие зависимости; их делает один проход и **замораживает**. Только потом фан-аут.
- **Фан-аут — по листовым компонентам.** Один агент = один файл/компонент из §3 (F1). Каждый агент:
  1) читает §2 и §3-строку; 2) применяет токены/примитивы; 3) проверяет себя на `/_gallery`;
  4) гонит `typecheck`+`lint`.
- **Изоляция:** при параллельной правке многих файлов давать агентам `isolation: "worktree"`, чтобы не
  конфликтовать; общий `index.css`/`icons.tsx`/барели НЕ трогаются в F1 (заморожены в F0).
- **Анти-разнобой:** общий контракт (§2) + галерея + запрет ad-hoc стекла/уголков/теней. Финальный
  агент-критик: «найди компоненты, отклоняющиеся от контракта» → следующий раунд правок.
- **Верификация:** галерея + скриншоты в обоих размерах/состояниях; для страниц — превью-сервер.
